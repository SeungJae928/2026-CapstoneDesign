# PLANS.md

기준 날짜 및 시간: 2026-04-13 18:48:26 (Asia/Seoul)

## 목적
이 문서는 이 저장소에서 복잡한 작업을 시작하기 전에 작성하는 실행 계획의 기준 템플릿이다.

이 템플릿은 실제 작업별 계획 문서를 만들 때 복사해서 사용한다. 계획은 한 번 쓰고 끝나는 문서가 아니라, 진행 상황과 결정 사항을 반영하며 계속 갱신하는 living document로 다룬다.

## 사용 방식
- 복잡한 기능 추가, 추천/랭킹/배치 설계, 구조 영향이 큰 수정, 다단계 작업, 반나절 이상 걸릴 수 있는 작업에 사용한다.
- 계획 작성 후에는 바로 구현하지 않고, 사용자 검토 또는 승인을 받기 전까지는 분석과 문서화까지만 진행한다.
- 승인 후 구현을 시작하고, 진행 중에는 `Progress`와 `결정 사항 / 변경 로그`를 갱신한다.
- 작업 완료 후에는 검증 결과와 남은 이슈를 반영한다.
- 완료된 계획 문서는 `plans/archive/`로 이동해 보관한다.

2. 각 단계에서 무엇을 확인하고 무엇을 바꾸는지 적는다.
3. 문서 수정이 같이 필요한지 적는다.
